# Requirements & Implementation Plan: AInterview Features

This document outlines the planning and requirements for the **AI Interview Plan** (CRUD & scheduling) and **AI Interview system** (HR and Technical stages for Intern, Junior, and Senior Mobile Programmers).

> IMPORTANT:
> DEVELOPER CONSTRAINTS:
> - Do NOT modify any existing files inside `lib/constants` or `lib/widgets`.
> - Use the existing widgets and design constants as they are. New screens, state management logic, services, and features should be implemented in other directories (e.g., `lib/screens`, `lib/services`, `lib/models`, `lib/providers`).

---

## 1. Project Requirements

### A. AI Interview Plan (CRUD & Scheduling)
User can schedule and manage their interview prep timeline. The system will automatically generate a custom preparation plan leading up to the target interview date.
- **Add Plan**: Select an interview date, target level (Intern, Junior, Senior), and language (e.g., English, Indonesian).
- **View Plan**: Display a timeline/schedule of preparation tasks (mock interviews, specific topic study guides) dynamically generated by the AI or rule-based templates matching the user's level and remaining prep days.
- **Edit Plan**: Allow updating the target date and level, which recalculates the timeline.
- **Delete Plan**: Delete the prep schedule.
- **Firestore Integration**: Persist plans in a `user_interview_plans` subcollection under the authenticated user's document.

### B. AI Interview Chatbot (Mobile Programmer)
A voice/text-based conversational agent mimicking a real job interview.
- **Three Levels**:
  - **Intern**: Focus on core programming concepts, fundamental data structures, basic OOP, and platform basics (Flutter/Dart or Android/iOS lifecycle).
  - **Junior Dev**: Focus on state management, APIs/networking, database integration, Git version control, and debugging.
  - **Senior Dev**: Focus on architecture (Clean Architecture, MVVM), system design, optimization, testing strategies, security, and team collaboration scenarios.
- **Two Stages per Level**:
  - **Stage 1: HR / Behavioral Interview**: General background, problem-solving soft skills, communication, fit, and motivation.
  - **Stage 2: Technical / User Interview**: Deep dive into coding concepts, architecture, and live scenario solving.
- **AI Engine**: Powered by Gemini API (or equivalent AI integration), configured with distinct system instructions for each stage and level.
- **Voice Capabilities**: Integration with speech-to-text (STT) for user input and text-to-speech (TTS) for the interviewer's voice responses to create a realistic mock interview environment.

## 3. UI/UX Flow & Screen Design

1. **Dashboard / Home Navigation**:
   - Add a tab or section for **Interview Plan** showing the current upcoming plan countdown and progress.
   - Add a quick start button for **AI Interview Chatbot**.
2. **Plan Form Screen**:
   - Date picker, Dropdowns for Level and Language.
   - Button to "Generate AI Practice Plan".
3. **Plan Detail Timeline Screen**:
   - Lists structured tasks day-by-day.
   - Option to mark tasks as completed.
   - Buttons to edit or delete the plan.
4. **AI Interview Setup & Lobby**:
   - Select Level (Intern / Junior / Senior) and Stage (HR / Technical).
   - Preview guidelines and microphone status.
5. **AI Interview Session Screen**:
   - Clean, immersive chat interface showing the AI interviewer profile.
   - Voice toggle / push-to-talk button using STT.
   - Text display showing transcripts of the conversation.
   - "End Interview & Get Review" button.
